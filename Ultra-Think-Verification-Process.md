# System Architect Interview: Ultra-Think Verification Process

## 1. Critical Decision Framework

### 1.1 Decision Validation Process

**Ultra-Think Checklist**
```
1. Challenge every assumption
2. Question the goal: Is this the RIGHT problem?
3. Stress-test: What happens when things go wrong?
4. Identify blind spots: What haven't we considered?
5. Expose circular reasoning and hidden dependencies
6. Validate against edge cases
7. Consider long-term implications
8. Assess organizational readiness
9. Evaluate alternative approaches
10. Document architectural trade-offs
```

**Decision Quality Assessment**
```
1. Business alignment
2. Technical feasibility
3. Financial viability
4. Operational sustainability
5. Security compliance
6. Scalability potential
7. Maintainability
8. Risk management
9. Stakeholder impact
10. Time-to-market considerations
```
